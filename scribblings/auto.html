Autobinding Module

 (require jrffi/auto)

(jrequire java-class ... class-option ...)
 
class-option = 
  | #:import
  | #:racketify
  | #:renamer renamer-proc
  | #:typemap typemap-proc
 
  java-class : id?
  renamer-proc : (type string? string? . -> . string?)
  type : (or/c 'predicate 'class 'mutator 'accessor 'constructor 'method)
Unpacks the bindings for the jrequired java classes. For each class the following are binded:

  • java-class, the name of the java-class is binded to the java class’ id

  • new-java-class, it is binded only if the java-class has a public constructor

  • java-class?, takes in an object and checks if it is an instance of java-class

  • java-class-method, it is binded for each method, if the method is overloaded

  • get-java-class-field, for each public java-class-field; an accessor procedure is binded, if the java-class-field is static the accessor does not take any arguments, if the java-class-field is non-static the accessor takes in an instance object of type java-class

  • set-java-class-field!, for each public non-final java-class-field a mutator procedure is binded, if the java-class-field is static the mutator takes a new field value, if the java-class-field is non-static the mutator takes in an instance object of type java-class and a new field-value

If the #:import option is specified the java class namespace is stripped from the resulting generated names.

Note, the #:racketify and the #:renamer renamer-proc options cannot be used at the same time

If the #:racketify option is specified:

  • the word "to" is converted to "->"

  • the position before the beginning of a change in character case, will become split up by a hyphen

  • all characters are lower-cased

  • dots are converted to hyphens

  • "make-" instead of "new-" is used in the constructor name

For the #:renamer renamer option to be used, the renamer function must be present for-syntax. The provided renamer will be passed by the autobinder a symbol representing the type being one of object to be binded (’predicate ’class ’mutator, ’accessor, ’constructor, or ’method). The renamer is expected to return a string which will be used by the autobinder to bind the object. a string for class-name, a string representing the method or field name or #f if type ’class or ’predicate is passed.

> (require jrffi jrffi/auto)
> (jrequire java.lang.Integer #:import #:racketify)
> (integer->string (make-integer 100))

"100"

> (get-integer-max-value)

2147483647

> (integer? (make-integer 100))

#t
